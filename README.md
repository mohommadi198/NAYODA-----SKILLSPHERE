# SkillSphere — Frontend

The **SkillSphere** frontend is a React single-page application (SPA) for a freelance
marketplace where clients post gigs and freelancers submit proposals, chat in real time,
and collaborate inside project workspaces. It is built with Create React App and talks to
the Express + MongoDB backend in `../backend`.

> This README replaces the default Create React App boilerplate and documents the
> **actual** SkillSphere project.

---

## Tech Stack

| Concern            | Tooling                                              |
| ------------------ | ---------------------------------------------------- |
| Framework          | React 19 + React DOM                                 |
| Build tool         | Create React App (`react-scripts` 5.0.1)             |
| Routing            | React Router v7 (`react-router-dom`)                 |
| Styling            | Tailwind CSS 3 + PostCSS + Autoprefixer              |
| Animation          | Framer Motion (`motion`) + `lucide-react` / `react-icons` |
| HTTP client        | Axios (with auth interceptor)                        |
| Auth (identity)    | Firebase Auth (`firebase`)                           |
| Real-time chat     | Socket.IO client (`socket.io-client`)               |
| Testing            | React Testing Library + Jest (via CRA)               |

## Prerequisites

- **Node.js** 18+ (LTS recommended)
- **npm** (ships with Node)
- A running instance of the **SkillSphere backend** (see `../backend/server.js`)
- A **MongoDB** connection string
- A **Firebase** web app project (for authentication)

## Environment Variables

Create a `.env` file in this folder (copy from `.env.example` if present). The app reads
these at build time via `process.env.REACT_APP_*`:

```dotenv
# ── Backend API ───────────────────────────────────────────────
# Base URL the Axios client (src/services/api.js) sends requests to.
REACT_APP_API_URL=http://localhost:5000/api

# ── Firebase (src/configs/firebaseConfig.js) ──────────────────
REACT_APP_FIREBASE_API_KEY=your_api_key
REACT_APP_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
REACT_APP_FIREBASE_PROJECT_ID=your_project_id
REACT_APP_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
REACT_APP_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
REACT_APP_FIREBASE_APP_ID=your_app_id
REACT_APP_FIREBASE_MEASUREMENT_ID=your_measurement_id
```

> **Note:** Variables must be prefixed with `REACT_APP_` to be exposed to the browser
> bundle. Restart `npm start` after changing `.env`.

The backend expects its own `.env` (in `../backend`): `MONGO_URI`, `JWT_SECRET`, and
optionally `PORT` (defaults to `5000`).

## Getting Started

```bash
# 1. Install dependencies
npm install

# 2. Configure environment (see above)
cp .env.example .env   # then fill in real values

# 3. Start the dev server
npm start
```

The app opens at [http://localhost:3000](http://localhost:3000) and reloads on change.

Make sure the backend is running and reachable at `REACT_APP_API_URL`, otherwise
API calls and real-time chat will fail.

## Routing & Pages

All routes live in `src/App.js` and are wrapped by `<Layout>` (Navbar + Footer).
Pages are lazy-loaded with `React.lazy` + `Suspense`.

| Path                          | Page                    | Access            |
| ----------------------------- | ----------------------- | ----------------- |
| `/`                           | Home                    | Public            |
| `/login`, `/signup`           | Login, Signup           | Public            |
| `/faq`, `/pricing`, `/blog`   | FAQ, Pricing, Blog      | Public            |
| `/browse/jobs`                | BrowseJobs              | Public            |
| `/browse/freelancers`        | BrowserFreelancer       | Public            |
| `/dashboard`                  | Dashboard               | Authenticated     |
| `/profile`, `/edit-profile`   | Profile, EditProfile    | Authenticated     |
| `/choose-role`                | RoleSelect              | Authenticated     |
| `/chat`                       | RealTimeChat            | Authenticated     |
| `/payments`                   | PaymentsFinances        | Authenticated     |
| `/project-workspace/:jobId`   | ProjectWorkspace        | Authenticated     |
| `/client/dashboard`           | ClientDashboard         | Client role       |
| `/create-gig`                 | CreateGig               | Client role       |
| `/freelancer/dashboard`       | FreelancerDashboard     | Freelancer role   |
| `/submit-proposal/:projectId` | SubmitProposal          | Freelancer role   |
| `/admin`                      | AdminDashboard          | Admin only        |
| `*`                           | NotFound                | 404 catch-all     |

Route protection is handled by `<ProtectedRoute>` (auth + optional `requireRole` /
`adminOnly` props) in `src/components/ProtectedRoute.jsx`, which relies on the role
helpers in `src/utils/roles.js`.

## Authentication

- **Identity:** Firebase Auth (see `src/Context/AuthContext.js` and `src/configs/firebaseConfig.js`).
- **API authorization:** After sign-in, the app stores a JWT in
  `localStorage["skillsphere_token"]`. `src/services/api.js` attaches it as a
  `Authorization: Bearer <token>` header on every request, and the Socket.IO client
  reuses the same token to authenticate the real-time connection.

---

## Real-time Chat

`src/Context/ChatContext.js` opens a Socket.IO connection to the backend. The backend
authenticates the socket with the same JWT used by the REST API and tracks online users
in an `onlineUsers` map (see `../backend/socket/chatSocket.js`).

---

## Connecting to the Backend

The frontend consumes the REST API mounted in `../backend/server.js`:

```
/api/users      → usersRouter
/api/proposals  → proposalRouter
/api/jobs       → jobsRouter
/api/chat       → chatRouter
```

Service wrappers in `src/services/*` centralize these calls, and `src/services/api.js`
provides the configured Axios instance used by all of them.
